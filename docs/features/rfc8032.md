# RFC 8032 — Edwards-Curve Digital Signature Algorithm (EdDSA)

(`tasks/specs/rfc8032.txt`, not in git). Each requirement carries the test

## §5.1 Ed25519 Parameters

- [x] 8032-001 (§5.1) The implementation shall instantiate EdDSA with the
  Ed25519 parameters: b = 256, curve edwards25519 over GF(2^255 - 19) with
  a = -1 and d = -121665/121666, base point B, and group order
  L = 2^252 + 27742317777372353535851937790883648493.
  - test: `tests/crypto/ed25519_sign_test.c` — `test_ed25519_sign_test1`
  - test: `tests/crypto/ed25519_sign_test.c` — `test_ed25519_sign_test2`
- [x] 8032-002 (§5.1) The implementation shall use SHA-512 as the hash
  function H.
  - test: `tests/crypto/ed25519_test.c` — `test_sha512_empty`
- [x] 8032-003 (§5.1) For Ed25519, the implementation shall prepend no dom2
  prefix (dom2(f, c) is the empty string) and accept no context.
  - test: `tests/crypto/ed25519_sign_test.c` — `test_ed25519_sign_test1`

## §5.1.2 Encoding

- [x] 8032-004 (§5.1.2) The implementation shall encode a point as the
  32-octet little-endian y-coordinate with the least significant bit of the
  x-coordinate copied into the most significant bit of the final octet.
  - test: `tests/crypto/ed25519_sign_test.c` — `test_ed25519_sign_test1`
  - test: `tests/crypto/ed25519_sign_test.c` — `test_ed25519_sign_test3`

## §5.1.3 Decoding

- [x] 8032-005 (§5.1.3) The implementation shall decode a 32-octet point by
  clearing bit 255 to obtain y, recovering x from the candidate root
  u*v^3*(u*v^7)^((p-5)/8) with the sqrt(-1) correction, and selecting the
  root whose parity matches x_0.
  - test: `tests/crypto/ed25519_test.c` — `test_ed25519_rfc_test1`
  - test: `tests/crypto/ed25519_test.c` — `test_ed25519_rfc_test2`
- [ ] 8032-006 (§5.1.3) If the decoded y value is >= p, then the
  implementation shall fail the decoding.
  - gap: `quic_ed_ge_decode` reduces a non-canonical y modulo p instead of
    rejecting it; no test.
- [ ] 8032-007 (§5.1.3) If no square root exists modulo p, or x = 0 while
  x_0 = 1, then the implementation shall fail the decoding.
  - gap: implemented in `quic_ed_ge_decode`, but no test feeds an
    undecodable encoding.

## §5.1.4 Point Addition

- [x] 8032-008 (§5.1.4) The implementation shall represent points in
  extended homogeneous coordinates (X, Y, Z, T) and add them with the

  - test: `tests/crypto/ed25519_sign_test.c` — `test_ed25519_sign_test1`

## §5.1.5 Key Generation

- [x] 8032-009 (§5.1.5) The implementation shall derive the public key by
  hashing the 32-byte private key with SHA-512, pruning the lower half
  (clear the lowest three bits of the first octet, clear the highest bit and
  set the second-highest bit of the last octet), and encoding [s]B.
  - test: `tests/crypto/ed25519_sign_test.c` — `test_ed25519_sign_test1`
  - test: `tests/crypto/ed25519_sign_test.c` — `test_ed25519_sign_test2`
  - test: `tests/crypto/ed25519_sign_test.c` — `test_ed25519_sign_test3`

## §5.1.6 Sign

- [x] 8032-010 (§5.1.6) When signing a message M, the implementation shall
  compute r = SHA-512(prefix || M) interpreted as a little-endian integer
  (reduced modulo L) and R as the encoding of [r]B.
  - test: `tests/crypto/ed25519_sign_test.c` — `test_ed25519_sign_test1`
  - test: `tests/crypto/ed25519_sign_test.c` — `test_ed25519_sign_test3`
- [x] 8032-011 (§5.1.6) The implementation shall compute
  k = SHA-512(R || A || M) mod L and S = (r + k * s) mod L, and emit the
  signature as R (32 octets) followed by the little-endian encoding of S
  (32 octets).
  - test: `tests/crypto/ed25519_sign_test.c` — `test_ed25519_sign_test1`
  - test: `tests/crypto/ed25519_sign_test.c` — `test_ed25519_sign_test2`

## §5.1.7 Verify

- [x] 8032-012 (§5.1.7) When verifying, the implementation shall split the
  signature into a 32-octet R and a 32-octet S and decode R and the public
  key A as points.
  - test: `tests/crypto/ed25519_test.c` — `test_ed25519_rfc_test1`
- [x] 8032-013 (§5.1.7, §8.4) If S is not in the range 0 <= S < L, then the
  implementation shall reject the signature.
  - test: `tests/crypto/ed25519_sign_test.c` —
    `test_ed25519_verify_s_eq_l_rejected`
- [x] 8032-014 (§5.1.7) The implementation shall check the group equation
  [S]B = R + [k]A' with k = SHA-512(R || A || M) mod L, accepting the
  signature only when it holds.
  - test: `tests/crypto/ed25519_test.c` — `test_ed25519_rfc_test1`
  - test: `tests/crypto/ed25519_test.c` — `test_ed25519_rfc_test3`
- [x] 8032-015 (§5.1.7) If the signature does not satisfy the group
  equation, then the implementation shall reject it.
  - test: `tests/crypto/ed25519_test.c` — `test_ed25519_rfc_test1`
  - test: `tests/crypto/ed25519_test.c` — `test_ed25519_rfc_test2`
  - test: `tests/crypto/ed25519_test.c` — `test_ed25519_rfc_test3`
