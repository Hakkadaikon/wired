# RFC 9287 — Greasing the QUIC Bit

EARS requirement ledger extracted from the spec text
(`tasks/specs/rfc9287.txt`, not in git). Each requirement carries the test
that demonstrates it; an unchecked box with no test line is an open gap.
Status as of 2026-07.

Legend:

- `[x]` — demonstrated by the referenced test
- `[~]` — exercised indirectly (evidence line explains how; no dedicated test)
- `[ ]` — not demonstrated by any test yet

**Coverage: 6/9 tested, 3 indirect, 0 untested.**

## §3 The Grease QUIC Bit Transport Parameter

- [x] 9287-001 (§3) Where greasing is enabled, the server shall send the
  grease_quic_bit transport parameter (0x2ab2) with an empty value.
  - test: `tests/tls/grease_test.c` — `test_grease_param`
- [x] 9287-002 (§3) If a grease_quic_bit transport parameter with a
  non-empty value is received, then the server shall treat it as a
  connection error of type TRANSPORT_PARAMETER_ERROR.
  - test: `tests/tls/grease_test.c` — `test_grease_param`
- [x] 9287-003 (§3) While the server advertises the grease_quic_bit
  transport parameter, the server shall accept packets with the QUIC Bit
  (0x40) set to 0.
  - test: `tests/tls/grease_test.c` — `test_grease_accept`

## §3.1 Clearing the QUIC Bit

- [x] 9287-004 (§3.1) The server shall set the QUIC Bit to 0 only when the
  peer advertised the grease_quic_bit transport parameter.
  - test: `tests/tls/bitset_test.c` — `test_bitset_may_clear`
  - test: `tests/tls/bitset_test.c` — `test_bitset_apply`
- [~] 9287-005 (§3.1) When the peer advertised grease_quic_bit, the server
  should set the QUIC Bit to an unpredictable value.
  - evidence: the set/clear mechanics are tested
    (`test_bitset_apply` leaves the other header bits intact), but no test
    asserts the chosen value is randomized across packets.
- [x] 9287-006 (§3.1) The server shall set the QUIC Bit to 0 only after
  processing the client's transport parameters and advertising
  grease_quic_bit itself.
  - test: `tests/tls/early_test.c` — `test_early_server`
- [~] 9287-007 (§3.1) The server shall not set the QUIC Bit to 0 based on a
  remembered negotiation from a previous connection.
  - evidence: the gate (`test_early_server`) requires the
    current connection's transport parameters to have been processed; no
    dedicated cross-connection memory test exists.
- [~] 9287-008 (§3.1) Where acting as a client, the implementation shall
  not set the QUIC Bit to 0 in early packets unless its Initial packets
  carry a token from a NEW_TOKEN frame received less than 604800 seconds
  prior on a connection where the server also advertised grease_quic_bit.
  - evidence: `test_early_client` in `tests/tls/early_test.c` gates early
    clearing on remembered server grease support; the NEW_TOKEN presence
    and 7-day age conditions have no dedicated test.
- [x] 9287-009 (§3.1) The server shall accept any QUIC Bit value on a
  Stateless Reset packet, since endpoints cannot know whether the peer can
  grease after connection state is lost.
  - test: `tests/tls/sreset_bit_test.c` — `test_sreset_bit_any`

## Out of scope

Requirements on parties or features this server SDK does not implement,
excluded from the coverage denominator:

- (§3.2) Extensions that assign semantics to the QUIC Bit and their
  negotiation rules — no such extension is shipped.
- (§4) The effect of greasing on network operators' traffic classification
  — operator consideration, no SDK code.
