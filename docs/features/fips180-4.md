# FIPS 180-4 — Secure Hash Standard (SHS)

EARS requirement ledger extracted from the spec text
(`tasks/specs/fips180-4.pdf`, not in git), scoped to the hash functions the
SDK ships: SHA-256 (`src/crypto/symmetric/hash/hash/sha256.c`), SHA-384 and
SHA-512 (`sha384.c`/`sha512.c`, sharing the SHA-512 compression). SHA-1,
SHA-224, SHA-512/224 and SHA-512/256 are not shipped. Each requirement
carries the test that demonstrates it; an unchecked box with no test line is
an open gap. Status as of 2026-07.

Legend:

- `[x]` — demonstrated by the referenced test
- `[~]` — exercised indirectly (evidence line explains how; no dedicated test)
- `[ ]` — not demonstrated by any test yet

**Coverage: 23/25 tested, 1 indirect, 1 untested.**

## §3.2 Operations on Words

## §4.1.2 SHA-224 and SHA-256 Functions

- [x] F180-003 (§4.1.2) The implementation shall compute
  Ch(x,y,z) = (x AND y) XOR (NOT x AND z) over 32-bit words.
  - test: `tests/crypto/sha256_test.c` — `test_sha256_vectors`
- [x] F180-004 (§4.1.2) The implementation shall compute
  Maj(x,y,z) = (x AND y) XOR (x AND z) XOR (y AND z) over 32-bit words.
  - test: `tests/crypto/sha256_test.c` — `test_sha256_vectors`
- [x] F180-005 (§4.1.2) The implementation shall compute the two capital-sigma
  functions for 32-bit words as
  Sigma0(x) = ROTR2(x) XOR ROTR13(x) XOR ROTR22(x) and
  Sigma1(x) = ROTR6(x) XOR ROTR11(x) XOR ROTR25(x).
  - test: `tests/crypto/sha256_test.c` — `test_sha256_vectors`
- [x] F180-006 (§4.1.2) The implementation shall compute the two lower-case
  sigma functions for 32-bit words as
  sigma0(x) = ROTR7(x) XOR ROTR18(x) XOR SHR3(x) and
  sigma1(x) = ROTR17(x) XOR ROTR19(x) XOR SHR10(x).
  - test: `tests/crypto/sha256_test.c` — `test_sha256_vectors`

## §5.1 Padding the Message

- [x] F180-012 (§5.1.1) For SHA-256, the implementation shall pad a
  message of length l bits by appending a single "1" bit, then k zero bits
  where k is the smallest non-negative solution to l+1+k = 448 (mod 512),
  then the 64-bit big-endian message length.
  - test: `tests/crypto/sha256_test.c` — `test_sha256_vectors`
- [x] F180-013 (§5.1.2) For SHA-384/SHA-512, the implementation shall pad a
  message of length l bits by appending a single "1" bit, then k zero bits
  where k is the smallest non-negative solution to l+1+k = 896 (mod 1024),
  then the 128-bit big-endian message length.
  - test: `tests/crypto/sha384_test.c` — `test_sha384_pad_boundary`

## §6.2 SHA-256 Hash Computation

- [x] F180-019 (§6.2.2) The implementation shall prepare the 64-word message
  schedule W_t: for 0<=t<=15, W_t is the corresponding message block word;
  for 16<=t<=63,
  W_t = sigma1(W_{t-2}) + W_{t-7} + sigma0(W_{t-15}) + W_{t-16} (mod 2^32).
  - test: `tests/crypto/sha256_test.c` — `test_sha256_vectors`
- [x] F180-020 (§6.2.2) The implementation shall initialize the eight
  working variables a..h from the previous hash value H(i-1) and run the
  64-round compression function using Ch, Maj, Sigma0, Sigma1, K_t and W_t,
  updating a..h each round.
  - test: `tests/crypto/sha256_test.c` — `test_sha256_vectors`
- [x] F180-021 (§6.2.2) The implementation shall compute the ith
  intermediate hash value H(i) by adding each working variable to the
  corresponding word of H(i-1) modulo 2^32, and shall output H(N) as the
  256-bit message digest after the last block.
  - test: `tests/crypto/sha256_test.c` — `test_sha256_vectors`
  - test: `tests/crypto/sha256_test.c` — `test_sha256_streaming`

## §6.4 SHA-512 Hash Computation

- [x] F180-022 (§6.4.2) The implementation shall prepare the 80-word message
  schedule and run the 80-round compression function over 64-bit words with
  the SHA-512 Sigma/sigma functions and K_t constants, the same structure as
  SHA-256 but with w=64 and 80 rounds.
  - test: `tests/crypto/ed25519_test.c` — `test_sha512_empty`
- [~] F180-023 (§6.4.2) The implementation shall process each padded
  1024-bit message block in order and, after the final block, output H(N)
  as the 512-bit message digest.
  - evidence: `quic_ed25519_sign`/`quic_ed25519_verify` stream two-plus-block
    SHA-512 inputs (32-byte prefix + message) through `quic_sha512_update`
    across the RFC 8032 test vectors in `tests/crypto/ed25519_test.c`; no
    dedicated SHA-512 multi-block KAT with a pinned digest exists.

## §6.5 SHA-384

- [x] F180-024 (§6.5) The implementation shall compute SHA-384 identically
  to SHA-512 except for using the SHA-384 initial hash value and truncating
  the final 512-bit hash value to its left-most 384 bits.
  - test: `tests/crypto/sha384_test.c` — `test_sha384_vectors`
  - test: `tests/crypto/sha384_test.c` — `test_sha384_pad_boundary`

## §7 Truncation of a Message Digest

- [ ] F180-025 (§7) Where an application requires a message digest length
