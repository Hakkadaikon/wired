# RFC 5280 — Internet X.509 PKI Certificate and CRL Profile

EARS requirement ledger extracted from the spec text
(`tasks/specs/rfc5280.txt`, not in git). Each requirement carries the test
that demonstrates it; an unchecked box with no test line is an open gap.
Scope: what this TLS server SDK consumes — certificate DER parsing and
certification path validation. Status as of 2026-07.

Legend:

- `[x]` — demonstrated by the referenced test
- `[~]` — exercised indirectly (evidence line explains how; no dedicated test)
- `[ ]` — not demonstrated by any test yet

**Coverage: 26/44 tested, 5 indirect, 13 untested.**

## Out of scope

Client-only, CA/operator-level, or deliberately unshipped requirements this
server SDK does not implement, excluded from the coverage denominator:

- (§2–§3) Requirements, assumptions, and overview of approach —
  informational.
- (§3.3, §5, §5.2, §5.3) CRL profile, CRL extensions, and CRL entry
  extensions — no CRL code in `src/`.
- (§6.1.3 (a)(3), §6.3) Revocation status determination and CRL validation —
  no revocation mechanism shipped.
- (§4.1.2.2, §4.1.2.4, §4.1.2.5, §4.1.2.6, §4.2) Conforming-CA issuance
  requirements (serial assignment, DirectoryString encoding choices,
  UTCTime-through-2049 encoding, criticality markings) — CA/operator
  behavior, not certificate consumption.
- (§4.2.1.5) Policy mappings — optional application feature, not shipped.
- (§4.2.1.7) Issuer alternative name — not processed in path validation per
  the spec itself.
- (§4.2.1.8) Subject directory attributes — no relying-party processing
  requirement.
- (§4.2.1.13, §4.2.1.15) CRL distribution points and freshest CRL — CRL
  retrieval machinery, not shipped.
- (§4.2.2.1, §4.2.2.2) Authority and subject information access (OCSP,
  caIssuers) — no online retrieval code.
- (§4.2.1.6) Wildcard name semantics — not addressed by RFC 5280; delegated
  to the RFC 6125 host-identity rules.
- (§4.2.1.6) Matching of non-dNSName SAN forms (rfc822Name, iPAddress,
  URI) — the SDK consumes DNS hostname identities only.
- (§7.2–§7.5) Internationalized domain names, IRIs, and mail addresses —
  no IDN processing consumed.
- (§8) Security considerations — informational guidance.
