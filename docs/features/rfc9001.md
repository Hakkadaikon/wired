## Out of scope

Client-only or operator-level requirements this server SDK does not
implement, excluded from the coverage denominator:

- (¬ß4.1.2) Client confirmation of the handshake on HANDSHAKE_DONE receipt or
  a 1-RTT acknowledgment ‚Äî client behavior.
- (¬ß4.1.4) Continuing to respond while asynchronous TLS processing completes
  ‚Äî the TLS stack runs synchronously in-process; no waiting state exists.
- (¬ß4.2) "Clients MUST NOT offer TLS versions older than 1.3" ‚Äî client
  behavior.
- (¬ß4.3) ClientHello size management (Retry-gated buffering of partial
  ClientHellos, minimizing ticket size) ‚Äî MAY-level operational guidance.
- (¬ß4.4) Client authentication of the server identity ‚Äî client behavior.
- (¬ß4.4) Client rejection of post-handshake CertificateRequest messages ‚Äî
  client behavior.
- (¬ß4.5) "Clients SHOULD NOT reuse tickets" ‚Äî client behavior.
- (¬ß4.6.1) Client rejection of a NewSessionTicket whose max_early_data_size
  is not 0xffffffff ‚Äî client behavior.
- (¬ß4.6.2) Client stream-state reset on 0-RTT rejection and treating a 0-RTT
  acknowledgment after rejection as an error ‚Äî client behavior.
