# RFC 7748 — Elliptic Curves for Security

EARS requirement ledger extracted from the spec text
(`tasks/specs/rfc7748.txt`, not in git). Each requirement carries the test
that demonstrates it; an unchecked box with no test line is an open gap.
Status as of 2026-07.

Legend:

- `[x]` — demonstrated by the referenced test
- `[~]` — exercised indirectly (evidence line explains how; no dedicated test)
- `[ ]` — not demonstrated by any test yet

**Coverage: 11/15 tested, 1 indirect, 3 untested.**

## §4.1 Curve25519

- [x] 7748-001 (§4.1) The implementation shall operate on curve25519, the
  Montgomery curve v^2 = u^3 + 486662*u^2 + u over GF(2^255 - 19).
  - test: `tests/tls/x25519_test.c` — `test_x25519_rfc_v1`
  - test: `tests/tls/x25519_test.c` — `test_x25519_rfc_v2`

## §5 The X25519 Function

- [x] 7748-002 (§5) The implementation shall take a 32-byte scalar and a
  32-byte u-coordinate as inputs and produce the 32-byte u-coordinate of the
  scalar multiple as output.
  - test: `tests/tls/x25519_test.c` — `test_x25519_rfc_v1`
  - test: `tests/tls/x25519_test.c` — `test_x25519_rfc_v2`
- [x] 7748-003 (§5) The implementation shall decode u-coordinates as
  little-endian integers.
  - test: `tests/tls/x25519_test.c` — `test_x25519_rfc_v1`
- [x] 7748-004 (§5) When receiving a u-coordinate, the implementation shall
  mask the most significant bit of the final byte.
  - test: `tests/tls/x25519_test.c` — `test_x25519_rfc_v2`
- [ ] 7748-005 (§5) The implementation shall accept non-canonical
  u-coordinate values (2^255 - 19 through 2^255 - 1 after masking) and
  process them as if reduced modulo p.
  - gap: the radix-2^51 field arithmetic reduces implicitly, but no test
    feeds a u-coordinate in [p, 2^255 - 1].
- [x] 7748-006 (§5) The implementation shall decode the scalar by clearing
  the three least significant bits of the first byte, clearing the most
  significant bit and setting the second most significant bit of the last
  byte, and reading the result little-endian.
  - test: `tests/tls/x25519_test.c` — `test_x25519_rfc_v1`
  - test: `tests/tls/x25519_test.c` — `test_x25519_rfc_v2`
- [x] 7748-007 (§5) The implementation shall compute the scalar multiple
  with the Montgomery ladder (a24 = 121665), one conditionally swapped
  ladder step per scalar bit, returning x_2 * z_2^(p - 2).
  - test: `tests/tls/x25519_test.c` — `test_x25519_rfc_v1`
  - test: `tests/tls/x25519_test.c` — `test_x25519_rfc_v2`
- [~] 7748-008 (§5) The implementation shall perform the conditional swap in
  constant time, independent of the swap argument.
  - evidence: `fe_cswap` (`src/tls/handshake/core/tls/x25519.c`) uses the
    spec's mask(swap) = 0 - swap arithmetic-mask XOR form with no branches;
    timing independence is not demonstrable by a unit test.
- [x] 7748-009 (§5) The implementation shall encode the resulting value as
  32 bytes in little-endian order with the unused most significant bit zero
  (fully reduced modulo p).
  - test: `tests/tls/x25519_test.c` — `test_x25519_rfc_v1`
  - test: `tests/tls/x25519_test.c` — `test_x25519_rfc_v2`

## §5.2 Test Vectors

- [x] 7748-010 (§5.2) The implementation shall produce the expected output
  u-coordinate for both X25519 input/output test vectors.
  - test: `tests/tls/x25519_test.c` — `test_x25519_rfc_v1`
  - test: `tests/tls/x25519_test.c` — `test_x25519_rfc_v2`
- [ ] 7748-011 (§5.2) The implementation shall produce the expected results
  for the iterated X25519 vectors (1, 1,000, and 1,000,000 iterations
  starting from k = u = the base point encoding).
  - gap: only the single-shot §5.2 vectors are pinned.

## §6.1 Diffie-Hellman

- [x] 7748-012 (§6.1) The implementation shall compute a public key as
  X25519(k, 9), where 9 is the base point u-coordinate encoded as the byte 9
  followed by 31 zero bytes.
  - test: `tests/tls/x25519_test.c` — `test_x25519_ecdhe`
- [x] 7748-013 (§6.1) When both parties apply their scalar to the peer's
  public key, the implementation shall arrive at the same shared secret
  K = X25519(a, X25519(b, 9)) = X25519(b, X25519(a, 9)).
  - test: `tests/tls/x25519_test.c` — `test_x25519_ecdhe`
- [x] 7748-014 (§6.1, §7) Where the computed output is the all-zero value
  (a small-order input point), the implementation shall detect it by ORing
  all output bytes and report failure.
  - test: `tests/tls/x25519_test.c` — `test_x25519_low_order_rejected`
- [ ] 7748-015 (§6.1) The implementation shall reproduce the §6.1
  Diffie-Hellman test vector (Alice's and Bob's key pairs and the shared
  secret K).
  - gap: `test_x25519_ecdhe` uses arbitrary scalars, not the pinned §6.1
    vector.

## Out of scope

Client-only or operator-level requirements this server SDK does not
implement, excluded from the coverage denominator:

- (§4.2, §5, §5.2, §6.2) Curve448 / X448 — not implemented; this SDK ships
  X25519 only.
- (§4.1) the birational maps to edwards25519 — informational; Ed25519 is
  covered by the [RFC 8032 ledger](rfc8032.md).
- (§5.1) general side-channel research guidance — informational advice; the
  cswap constant-time requirement itself is 7748-008.
- (§7) protocol-designer guidance (contributory-behavior assumptions,
  equivalent public keys, twist-rejecting implementations) — design
  guidance with no implementation behavior of its own.
- (Appendix A) the curve-derivation procedure — informational.
